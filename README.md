Carpeta destinada a almacenar toda la documentacion y codigos de los proyectos realizados por el club de robotica de la biblioteca EPM

Reglamento:
1. Todo código subido debe estar debidamente nombrado en el formato reto_mes_año_version, dejo un ajemplo a continuacion: Robot_sumo_marzo_2026_v2
2. Requisitos para competencias (documentación del robot a construir: esquemáticos, lista de materiales, códigos de pruebas unitarias, códigos finales con versiones,
   Robot completa el reto interno con consistencia ≥85% (5/6 corridas exitosas), Código modular y documentado: bitácora con métricas y fotos,
   Chasis estable y cableado limpio: plan de pruebas y roles definidos.
3. Reunión inicial SIEMPRE para alinear los objetivos a alcanzar en el día, determinar tiempos y evaluar estrategias para alcanzarlos (max 20 min).
4. Reunión de cierre SIEMPRE para analizar cumplimiento de objetivos, asignación de tareas asincrónicas (de ser necesario) y checklist para próxima sesión

Bitácora técnica (plantilla mínima)
1.	Objetivo del reto (una frase con verbo Bloom).
2.	Esquema/diagrama (foto o dibujo simple).
3.	Código (versión y cambios): qué y por qué.
4.	Métricas clave: tiempos, error, consistencia.
5.	Lecciones aprendidas y siguiente ajuste.


•	Sin documentación no hay materiales ni competencia. 
•	Roles por equipo: Líder técnico, Programador, Documentación/Pruebas (rotan en cada reto y cada grupo es responsable de saber en qué orden van a cambiar).
